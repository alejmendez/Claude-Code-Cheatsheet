Inicio Rápido Instalación y primeros pasos

  • claude updateActualizar a la última versión

Modo Headless / SDK Para CI/CD, scripts y automatización

📡 Print Mode (no interactivo)
  • claude -p "query"Ejecutar y salir (print mode)
  • claude -p --output-format json "query"Salida en JSON con metadata
  • claude -p --max-turns 5 "query"Limitar turnos de conversación
  • claude -c -p "check types"Continuar sesión en modo SDK
🔗 Piping y Scripting
# Piping de contenido a Claude cat logs.txt | claude -p "explica estos errores" git log --oneline | claude -p "resume estos commits" cat error.log | claude -p "encuentra la causa raíz" ls -la | claude -p "explica esta estructura" # Scripts de automatización claude -p "analiza codebase" \ --output-format json > analysis.json claude -p "genera tests" \ --max-turns 3 > tests.txt

Permisos y Seguridad Control granular de herramientas

✅ Permitir Herramientas
# Permitir herramientas específicas claude --allowedTools "Bash(git:*)" "Write" "Read" # Permitir solo lectura (git log, diff, grep...) claude --allowedTools \ "Bash(git log:*)" \ "Bash(git diff:*)" \ "Bash(grep:*)" \ "Bash(rg:*)" \ "Read"
🚫 Bloquear y Multi-directorio
# Bloquear herramientas peligrosas claude --disallowedTools \ "Bash(rm:*)" \ "Bash(sudo:*)" # Acceso a múltiples directorios claude --add-dir ../frontend ../backend ../shared # ⚠️ SOLO para sandboxes aislados sin internet claude --dangerously-skip-permissions

Thinking, Effort y Ultrathink Controla la profundidad de razonamiento de Claude

🎚️ /effort — Control de Profundidad (Recomendado)
  • /effort lowRápido y barato. Ideal para fixes de sintaxis, lint, refactors simples
  • /effort mediumBalance entre velocidad y calidad. Bueno para workflows diarios
  • /effort highPor defecto. Razonamiento profundo para tareas complejas
  • /effort maxMáxima capacidad. Solo disponible en Opus 4.6
# Configurar effort via variable de entorno CLAUDE_CODE_EFFORT_LEVEL=medium claude # O en settings (~/.claude/settings.json) { "effortLevel": "medium" } # Slider visual: aparece en /model con modelos compatibles
⚰️ Ultrathink — Deprecado (referencia histórica)
⚠️ Las keywords de thinking están DEPRECADAS desde enero 2026. Extended thinking ahora está habilitado por defecto. Usa /effort en su lugar para control granular. # Jerarquía antigua (ya no asignan tokens): "think" → budget mínimo "think hard" → budget medio "think harder" → budget alto "ultrathink" → budget máximo (31,999 tokens) # Ahora el equivalente es: /effort max → razonamiento máximo adaptativo # Para duplicar el budget de thinking (solo 64K output models): MAX_THINKING_TOKENS=63999 claude
🔄 Adaptive Thinking (Opus 4.6+)
  • TabToggle Thinking on / off en el REPL
  • Ctrl+OVerbose mode: ver el proceso de razonamiento interno
  • /modelCambiar modelo y ver slider de effort
  • /fastToggle Fast Mode (2.5x velocidad, 6x costo)
# Desactivar adaptive thinking (volver a budget fijo) CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING=1 claude # Con adaptive thinking, Claude piensa ENTRE tool calls: # Lee archivo → piensa → decide → llama herramienta → piensa otra vez # Tip: /effort low para tareas simples, ahorra 3-5x en tokens # Tip: /effort max para arquitectura y debugging complejo

Tips Pro para Productividad De junior a senior en cada prompt

1
Usa /compact frecuentemente para liberar contexto y mantener la calidad de respuestas alta
2
Shift+Tab+Tab → Plan Mode antes de tareas complejas. Claude planificará antes de escribir código
3
Inicia sesiones separadas para cada tarea. Ahorra tokens y mejora la calidad de las respuestas
4
Usa @filename para incluir archivos como contexto directo en tu prompt
5
Usa !`comando` para ejecutar bash inline antes de que Claude procese el resultado
6
Crea CLAUDE.md en la raíz de tu proyecto para dar contexto persistente y reglas al agente
7
~/.claude/CLAUDE.md para preferencias globales que aplican a todos tus proyectos
8
Monitorea costos con /cost y agrégalo a tu status line para nunca perder de vista el gasto
9
Usa git commits como checkpoints + /rewind para volver atrás si Claude toma un camino incorrecto
10
Extended Thinking se activa con Tab. Usa más tokens pero mejora drásticamente el razonamiento complejo
